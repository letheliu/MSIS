# 内网军队公文写作助手 - 设计文档

**日期**: 2026-04-19
**基于**: Sirchmunk 框架

## 一、需求概述

### 目标用户
- 多层级用户：文秘人员、军官/指挥员、普通官兵
- 需要分级权限管理系统

### 核心功能
1. 模板库与生成：提供公文模板，快速生成文档
2. 智能检索：基于Sirchmunk的检索能力，从内网文档库检索资料
3. 辅助校对：格式、用语、规范性检查
4. 协同编辑：多人协作编辑，版本控制，审批流程

### 公文类型
- 军事命令：命令、通令
- 行政公文：请示、报告、批复
- 内部文书：会议纪要、工作总结、简报
- 通用公文：通知、通报、函件

### 部署环境
- 完全离线内网环境
- 所有AI模型需要本地部署

### 开发优先级
- 快速验证：先实现模板库和智能检索

---

## 二、整体架构

### 分层架构

**数据层**
- 文档存储和索引管理
- Sirchmunk作为核心检索引擎
- 支持多种格式文档实时索引
- 本地文件系统 + SQLite/PostgreSQL存储

**服务层**
- 模板管理服务
- 智能检索服务（封装Sirchmunk API）
- 公文生成服务
- 辅助校对服务
- REST API接口

**用户层**
- Web前端界面（Vue.js/React）
- JWT令牌认证
- RBAC权限模型

---

## 三、模板库与生成模块

### 模板设计
- 结构化模板，包含：
  - 元数据：名称、类型、适用层级
  - 格式规范：字体、字号、行距、页边距
  - 动态字段：可替换的变量位置
- JSON格式存储

### 生成流程
1. 用户选择模板类型
2. 填写必要字段
3. 基于模板渲染初稿
4. 可选：调用智能检索补充背景资料
5. 生成最终文档

### 生成模型
- 轻量级本地模型：Qwen2.5-7B 或 DeepSeek-Coder
- 完全离线部署

---

## 四、智能检索模块

### Sirchmunk集成
- 部署为独立服务
- 无需向量嵌入，直接索引原始数据
- 降低离线环境资源要求

### 检索流程
1. 用户输入查询关键词
2. Sirchmunk实时检索
3. 返回相关文档片段和原文链接
4. 相关性排序展示
5. 支持按类型、日期、部门过滤

### 数据同步机制
- **批量导入**：扫描现有文档目录，批量索引
- **实时同步**：监控目录，新文件自动索引
- **增量更新**：检测文档修改，自动更新索引

### 检索增强
- 基于查询历史优化排序
- 用户反馈驱动自我进化

---

## 五、辅助校对模块

### 三层检查
1. **格式检查**：字体、字号、行距、页边距规范
2. **用语检查**：错别字、语法、非规范用语检测
3. **结构检查**：标题、主送机关、正文、发文机关、日期完整性

### 校对引擎
- 规则库：存储公文规范条目
- 轻量模型：语义层面检查（语气得体性）

---

## 六、协同编辑模块

### 实时编辑
- WebSocket实现实时同步
- 支持多人同时编辑

### 版本控制
- 记录每次修改
- 支持历史版本查看和回滚

### 审批流程
- 配置化流程设计
- 自定义审批节点和权限

### 文档状态流转
草稿 → 校对中 → 待审批 → 已发布

---

## 七、技术栈

### 后端
- Python 3.11+
- FastAPI（Web框架）
- SQLite/PostgreSQL（数据库）
- Redis（会话缓存，可选）

### 前端
- Vue 3
- Element Plus（UI组件）
- Pinia（状态管理）
- Axios（HTTP客户端）

### AI模型（本地部署）
- Sirchmunk：检索引擎
- Qwen2.5-7B-Instruct：文本生成
- vLLM/Ollama：推理加速

---

## 八、部署方案

### Docker Compose编排
- 单机部署：后端+前端+模型在同一服务器
- 分布式部署：模型服务独立，支持GPU加速

## 九、快速验证阶段（MVP）

### 目标
验证Sirchmunk基础能力和公文生成可行性

### 范围
1. 搭建Sirchmunk服务
2. 导入示例文档
3. 实现3-5个基础模板
4. 简单检索界面
5. 验证核心检索能力

### 不包含
